---
title: "Generate Speech"
description: "Generate speech with the REST TTS client or stream synthesized audio over WebSockets."
---

The SDK offers two TTS workflows. REST is the simpler choice when you want a complete audio file or raw buffer. WebSockets are better when you need incremental playback, interactive responses, or explicit flush control. The relevant implementations are `pkg/api/speak/v1/rest/speak.go` and `pkg/client/speak/v1/websocket/client_callback.go`.

<Steps>
<Step>
### Use REST when you want a file on disk

```go
ctx := context.Background()

dg := speakapi.New(speak.NewRESTWithDefaults())
resp, err := dg.ToSave(ctx, "hello.wav", "Hello from Deepgram", &interfaces.SpeakOptions{
  Model:      "aura-2-thalia-en",
  Encoding:   "linear16",
  SampleRate: 48000,
})
if err != nil {
  panic(err)
}

fmt.Println(resp.RequestID)
```

</Step>
<Step>
### Use WebSockets when you want incremental playback

```go
dg, err := speak.NewWSUsingCallback(ctx, "", &interfaces.ClientOptions{}, &interfaces.WSSpeakOptions{
  Model:      "aura-2-thalia-en",
  Encoding:   "linear16",
  SampleRate: 48000,
}, callback)
if err != nil {
  panic(err)
}

if dg.Connect() {
  _ = dg.SpeakWithText("Hello from Deepgram")
  _ = dg.Flush()
}
```

</Step>
</Steps>

## Complete Runnable REST Example

```go
package main

import (
  "context"
  "fmt"

  speakapi "github.com/deepgram/deepgram-go-sdk/v3/pkg/api/speak/v1/rest"
  "github.com/deepgram/deepgram-go-sdk/v3/pkg/client/interfaces"
  "github.com/deepgram/deepgram-go-sdk/v3/pkg/client/speak"
)

func main() {
  ctx := context.Background()
  dg := speakapi.New(speak.NewRESTWithDefaults())

  resp, err := dg.ToSave(ctx, "hello.wav", "Hello, this file was generated by Deepgram.", &interfaces.SpeakOptions{
    Model:      "aura-2-thalia-en",
    Encoding:   "linear16",
    SampleRate: 48000,
  })
  if err != nil {
    panic(err)
  }

  fmt.Println("saved:", resp.Filename)
  fmt.Println("request:", resp.RequestID)
  fmt.Println("characters:", resp.Characters)
}
```

## Complete Runnable Streaming Example

```go
package main

import (
  "context"
  "fmt"
  "os"

  speakapi "github.com/deepgram/deepgram-go-sdk/v3/pkg/api/speak/v1/websocket/interfaces"
  "github.com/deepgram/deepgram-go-sdk/v3/pkg/client/interfaces/v1"
  "github.com/deepgram/deepgram-go-sdk/v3/pkg/client/speak"
)

type Callback struct{}

func (Callback) Open(*speakapi.OpenResponse) error { return nil }
func (Callback) Metadata(*speakapi.MetadataResponse) error { return nil }
func (Callback) Flush(*speakapi.FlushedResponse) error { return nil }
func (Callback) Clear(*speakapi.ClearedResponse) error { return nil }
func (Callback) Close(*speakapi.CloseResponse) error { return nil }
func (Callback) Warning(*speakapi.WarningResponse) error { return nil }
func (Callback) Error(err *speakapi.ErrorResponse) error {
  fmt.Println("error:", err.ErrMsg)
  return nil
}
func (Callback) UnhandledEvent(by []byte) error { return nil }
func (Callback) Binary(by []byte) error {
  f, err := os.OpenFile("streamed.raw", os.O_CREATE|os.O_APPEND|os.O_WRONLY, 0o666)
  if err != nil {
    return err
  }
  defer f.Close()
  _, err = f.Write(by)
  return err
}

func main() {
  ctx := context.Background()
  dg, err := speak.NewWSUsingCallback(ctx, "", &interfaces.ClientOptions{}, &interfaces.WSSpeakOptions{
    Model:      "aura-2-thalia-en",
    Encoding:   "linear16",
    SampleRate: 48000,
  }, Callback{})
  if err != nil {
    panic(err)
  }

  if !dg.Connect() {
    panic("connect failed")
  }
  defer dg.Stop()

  if err := dg.SpeakWithText("Streaming text to speech with Deepgram."); err != nil {
    panic(err)
  }
  if err := dg.Flush(); err != nil {
    panic(err)
  }
}
```

The REST path is usually the right default. The WebSocket path is the better fit when your application needs to start playback before the full audio response exists.
